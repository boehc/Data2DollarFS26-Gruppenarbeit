Voraussetzungen:
  - Python 3.11+ installiert
  - Google Chrome installiert (für den Selenium-Scraper)
  - OpenAI API Key für Schritt 3

Virtuelle Umgebung erstellen und aktivieren:
  python -m venv .venv
  .venv\Scripts\activate          (Windows)
  source .venv/bin/activate       (Mac/Linux)

Basis-Pakete installieren (für Scraper + PDF-Extraktion):
  pip install -r requirements.txt

Inhalt requirements.txt:
  selenium>=4.6.0
  webdriver-manager>=3.8.0
  requests>=2.31.0
  pdfplumber>=0.10.0
  pandas>=2.0.0
  openpyxl>=3.1.0

Zusatzpaket für Scraper (undetected Chrome):
  pip install undetected-chromedriver

Zusatzpakete für Schritt 3 (Keyword-Extraktion via OpenAI):
  pip install -r requirements_skill_extraction.txt

Inhalt requirements_skill_extraction.txt:
  openai>=1.0.0
  pandas>=2.0.0
  tqdm>=4.65.0

OpenAI API Key setzen (einmalig pro Session):
  $env:OPENAI_API_KEY = "sk-..."    (Windows PowerShell)
  export OPENAI_API_KEY="sk-..."    (Mac/Linux)

SCHRITT 1 — MERKBLATT-PDFs HERUNTERLADEN (Scraper)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Welche Dateien: 0_Scraper_HS25.py / 0_Scraper_FS26.py
Benötigt:       Chrome-Browser, undetected-chromedriver, Internetverbindung

Ausführen:
  python 0_Scraper_HS25.py       ← Herbstsemester 2025
  python 0_Scraper_FS26.py       ← Frühjahrssemester 2026

Was passiert:
  - Browser öffnet courses.unisg.ch automatisch
  - Navigiert zu Master → 18MBIh → Fachstudium
  - Klappt alle Baumknoten auf (Pflichtbereich, Vertiefungen, …)
  - Sammelt alle Kurslinks und lädt für jeden Kurs das Merkblatt-PDF herunter
  - HS25-Scraper speichert in:  mbi_hs25_ohne_wahlbereich/   (54 PDFs)
  - FS26-Scraper speichert in:  FS26_mbi_ohne_wahlbereich/   (39 PDFs, gefiltert)
                       und in:  mbi_course_sheets_FS26_all/  (126 PDFs, alle)

WICHTIG — Ordnernamen:
  Der HS25-Scraper legt den Ordner mbi_hs25_ohne_wahlbereich/ an (Kleinbuchstaben).
  Das Skript 1_PDF_Extractor_to_CSV.py sucht aber nach HS25_mbi_ohne_wahlbereich/
  (anderer Name!). Entweder den Ordner umbenennen ODER in Schritt 2 den Pfad
  im Skript anpassen (Zeile ~80, Variable PDF_DIRECTORIES):

    PDF_DIRECTORIES = [
        Path("FS26_mbi_ohne_wahlbereich"),
        Path("mbi_hs25_ohne_wahlbereich"),   ← diesen Pfad korrekt setzen
    ]

Ergebnis: Zwei Ordner mit PDFs — je einer pro Semester.

SCHRITT 2 — PDFs IN CSV UMWANDELN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Welche Datei:  1_PDF_Extractor_to_CSV.py
Liest:         FS26_mbi_ohne_wahlbereich/  und  mbi_hs25_ohne_wahlbereich/
Gibt aus:      mbi_curriculum_database.csv  (dann manuell umbenennen, s.u.)

Ausführen:
  python 1_PDF_Extractor_to_CSV.py

Was passiert:
  - Liest alle PDFs aus beiden Ordnern mit pdfplumber
  - Erkennt automatisch Sprache (DE/EN)
  - Extrahiert pro Kurs:
      course_id, course_title, ects, language
      learning_objectives_raw  (Lernziele)
      course_content_raw       (Kursinhalte)
      mbi_profile              (eine der 6 Fachrichtungen)
  - Speichert als mbi_curriculum_database.csv (ca. ~94 Zeilen)

Nach dem Ausführen:
  Das resultierende mbi_curriculum_database.csv manuell prüfen und
  ggf. umbenennen zu mbi_curriculum_database_withSections.csv, damit
  Schritt 3 es einlesen kann.

  HINWEIS: Das Skript 2_extract_keywords_pipeline.py sucht nach:
    INPUT_FILE = "0_mbi_curriculum_database_withSections.csv"
  Die tatsächliche Datei heißt aber mbi_curriculum_database_withSections.csv.
  Entweder die Datei umbenennen (Präfix 0_ hinzufügen) ODER den INPUT_FILE-
  Wert in Schritt 3 anpassen.

Ergebnis: mbi_curriculum_database_withSections.csv mit allen Kurs-Metadaten.

SCHRITT 3 — KEYWORDS MIT OPENAI EXTRAHIEREN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Welche Datei:  2_extract_keywords_pipeline.py
Liest:         0_mbi_curriculum_database_withSections.csv
Gibt aus:      mbi_curriculum_enriched_keywords.csv
Braucht:       OpenAI API Key (Umgebungsvariable OPENAI_API_KEY)

Kosten: GPT-4o-mini ist sehr günstig (<1$ für ~94 Kurse).
        GPT-4o liefert bessere Ergebnisse, aber ist ca. 10–20x teurer.

Testlauf zuerst (nur 3 Kurse, um API-Key und Format zu prüfen):
  python 2_extract_keywords_pipeline.py --test

Vollständiger Produktionslauf:
  python 2_extract_keywords_pipeline.py

Mit besserem Modell:
  python 2_extract_keywords_pipeline.py --model gpt-4o

Was passiert:
  - Lädt das CSV mit allen Kursen
  - Pro Kurs: kombiniert Lernziele + Kursinhalte und schickt sie an OpenAI
  - OpenAI extrahiert englische Keywords (Hard Skills, Tools, Methoden, Domänen)
  - Keywords werden mit | (Pipe) getrennt gespeichert (kein Konflikt mit CSV-Semikolon)
  - Schreibt Checkpoint nach jeder Zeile → bei Abbruch kann weitergemacht werden

Checkpoint-Verhalten:
  Bei Unterbrechung (Strg+C) einfach neu starten — das Skript macht dort weiter,
  wo es aufgehört hat. Checkpoint liegt in: .keyword_extraction_checkpoint.json

Ergebnis: mbi_curriculum_enriched_keywords.csv
          (alle ursprünglichen Spalten + neue Spalte extracted_keywords)

Nach dem Ausführen:
  Das Skript 3_extract_startup_keywords.py liest 0.1_mbi_curriculum_enriched_keywords.csv.
  Datei also umbenennen oder den INPUT_FILE-Wert im Skript anpassen.
